## Decisions

- **`_partition_facts` returns `None` for spare slots** (facts < patterns) so those sub-batches keep the
  pre-42.28 fact-free prompt rather than an empty list — behavior-preserving fallback.
- **Case-insensitive topic dedup** — category="History" + prompt-token "history" would otherwise both
  survive and make Wikipedia search the same concept twice, re-introducing the near-duplicate facts the
  partition removes. (Caught by the 3-lens adversarial review.)
- **Filler stopwords added** ("fact", "facts", "interesting", "fun", "top", …) — "X fun facts about Y" is
  the dominant trivia-prompt shape; without these, filler ate the ≤3-token budget before the real topic.
- **OpenTDB no-topics path now issues 24 sequential HTTP calls (was 10)** — flagged, NOT fixed: it's free
  (no API cost) and now rare (prompt tokens almost always populate topics). Fixing it (early-break /
  sampling) would change fact-diversity behavior the plan didn't ask to touch.
- **42.26 must be SYNC for DedupStage** — `QuestionStore` Protocol declares `find_duplicates` sync and
  `DedupStage` calls it synchronously inside `except: return False` (async-called-sync = silent
  pass-through = dedup silently disabled). Do NOT use a separate sync DB engine; reuse the existing
  `_BackgroundLoop` (`run_coroutine_threadsafe`) pattern from
  `apps/quiz-agent/app/retrieval/sync_pgvector_store.py` (its `find_duplicates` is a `NotImplementedError`
  stub). See prior handoff (`handoff-2026-06-21-1928.md`) Decisions for the full 42.27 wiring plan.
