## Decisions

- **#46 iOS tail (B8/B9) stays Ralph-suitable, not `[HUMAN]`** — mba has the latest Xcode + macOS with iOS 26 SDK present (Xcode 26.3 staged no-admin, iOS 26.2 SDK), so there's no need to gate the iOS tasks. They ran and passed as part of the single A→B loop.
- **Comma excluded from deterministic split in 46.A2** — `"Tokyo, Japan"`, `"December 7, 1941"` etc. would be incorrectly split by a regex on comma. Unambiguous markers (`—`, `–`, `because`, `while`, `namely`, `due to`, `i.e.`) go through the deterministic path; comma-tailed answers go to the LLM normalization fallback with the explicit judgment question "canonical head + appositive, or one indivisible answer?".
- **PATH in tmux command string, not just launcher env** — setting `export PATH=...` in the launcher script doesn't propagate into the tmux pane shell (non-login, doesn't source `.zprofile`). Both launchers now set PATH explicitly inside the string passed to `tmux new -d -s ... "export PATH=... && cd ..."`.
- **#46 leads, #45 follows (sequential, not parallel)** — `ralph.sh` requires a clean working tree; running two loops on the same repo simultaneously would deadlock. Sequencing was: #46 backend+iOS → #45 iOS.
