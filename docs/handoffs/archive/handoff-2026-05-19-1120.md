**Stayed on primary Mac, deferred agent transition:** Aktívna debug iterácia (Xcode + sim + xcresult inšpekcia) je rýchlejšia na primary kde je všetko nakonfigurované. Agent Mac je pre overnight autonómne runy, nie pre interaktívnu prácu. SSH cez Claude Code by pridalo latenciu na každý xcodebuild cyklus.

**Diagnostický prístup — xcresult inšpekcia pred fix-and-rerun:** Namiesto trial-and-error fixov som extrahoval `App UI hierarchy` attachment z xcresult (`xcrun xcresulttool export object --legacy`). Ground truth a11y tree ukázal presný element type pre každý identifier — okamžite odhalil že `question.text` je `StaticText`, nie `Other`, a že container identifiers v `AnswerConfirmationView` prepisujú child identifiers. Bez tohto by som hádal cez viac iterácií.

**Removed `.accessibilityIdentifier("confirmation.state.transcript")` namiesto rename/refactor:** Marker identifiers neboli v žiadnom teste použité — surgical removal namiesto kompromisu (premenovať parent na niečo unique by stále spôsobilo collision s nejakým z children).

**Production-side fix (not test-side) pre `question.state`:** `.hidden()` na Text strhne label z a11y tree. Riešenie: odstrániť `.hidden()` a polagať na 0×0 frame. Alternatíva bola čítať state z `statusPill.value` — ale ten je už používaný a state probe bol vyrobený ako dedikovaný copy-independent seam, takže oprava production-side zachová pôvodný dizajn.
