## Notes & deviations from spec

- Step 6 used `label: "Cancel editing"` rather than `id: confirmation.editCancel` because the
  parent VStack's `.accessibilityIdentifier("confirmation.state.transcript")` propagates to child
  elements in the AX snapshot tree, shadowing `confirmation.editCancel`. The tap by label is
  functionally equivalent and the element was unambiguous (only one "Cancel editing" button exists).
  See "Followups" below for the fix.
- `question.state` probe and `question.statusPill` are not visible when the confirmation sheet
  overlays QuestionView — this is expected per skill spec. State was verified via log timeline.

## Followups (separate from RS-08 verdict)

1. **id shadowing by parent**: The AX snapshot shows all immediate children of the transcript-branch
   VStack inheriting `AXUniqueId: "confirmation.state.transcript"` because
   `.accessibilityIdentifier` on a VStack propagates down in UIKit's accessibility tree.
   Fix: wrap `transcriptBody` in an `.accessibilityElement(children: .contain)` container to
   isolate the parent id from child buttons, or apply `.accessibilityIdentifier` only to a
   zero-size spacer / hidden element rather than the VStack.
   Until fixed, automation steps must use `label:` for buttons inside `transcriptBody`.
2. **auto-confirm resubmit bug (issue-19)**: Separate concern tracked in
   `docs/issues/issue-19-auto-confirm-resubmit-bug.md`.
