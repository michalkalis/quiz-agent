## Notes & deviations from spec
- **Same edit-mechanic deviation as RS-06.** Cannot strictly replace the transcript: erasing `Paris` flips `isProcessing` true (`QuestionView.swift:63`) and removes the TextField mid-edit. Worked around by appending `Berlin` (field became `ParisBerlin`). Still satisfies the hypothesis вАФ the user is in edit mode with a modified transcript when they tap Re-record.
- **Auto-confirm disabled under --ui-test.** The 23-second window between Paris committing (14:30:49) and Re-record landing (14:31:12) would normally fire the auto-confirm timer (`Config.autoConfirmDelaySecs = 10`). With `MockPersistenceStore.savedSettings.autoConfirmEnabled = false`, the timer never starts, leaving the agent free to drive the edit-flow at accessibility-tree-tap pace. This is the precise unblock the previous RS-07 run on 2026-04-29 was waiting on.
- **Hypothesis-strengthening evidence.** `beginEditingTranscript()` cancels the auto-confirm timer; even if auto-confirm were enabled, this scenario would also be guarded by that cancel. Disabling at the settings layer is the simpler, less-flaky guard for the agent.
