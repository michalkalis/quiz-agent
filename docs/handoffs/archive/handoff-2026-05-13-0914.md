## Decisions

- **Environment mismatch stays inside `JWSInvalid` (not a fourth exception type).** Plan listed exactly 3 exceptions. Sandbox JWS hitting Production raises `JWSInvalid` with `"environment mismatch"` in the message ŌĆö distinct from bundle mismatch (`JWSWrongBundle`). Trade-off: acceptable for Phase 1 (one Fly app per environment, greppable message string); revisit in Phase 4 if multi-env routing makes `JWSInvalid` too noisy.
- **Apple root cert not checked in.** Repo `.gitignore` already blocked `*.cer`; a checked-in cert would make "is this current?" a code-review question. Download via `make fetch-apple-root`; CI wires that step in Task 1.12.
- **`from_settings` factory NOT added to verifier.** Keeps `Settings` import out of the verifier module so Task 1.9 wires its own FastAPI dep cleanly (e.g. `Depends(get_verifier)` factory function in the router or `app/dependencies.py`). Adding it now would be speculative.
- **Combined 1.7+1.8 commit** (`234df66`) rather than two separate commits ŌĆö plan file had 1.7+1.8 notes already merged, splitting would have needed `git add -p` gymnastics.
