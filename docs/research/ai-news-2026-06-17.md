## 1. OpenRouter Audio APIs — Whisper, Voxtral, and TTS now routable via unified gateway (June 12)

**Sources:**
- [New Audio APIs for Speech and Transcription — OpenRouter Blog](https://openrouter.ai/blog/announcements/announcing-audio-apis/)
- [Speech-to-Text models — OpenRouter](https://openrouter.ai/collections/speech-to-text-models)
- [Voxtral Mini Transcribe — OpenRouter](https://openrouter.ai/mistralai/voxtral-mini-transcribe)

**Summary:** OpenRouter's June 12 platform release added `/audio/transcriptions` and `/audio/speech` endpoints. STT models available: `openai/whisper-1`, `openai/whisper-large-v3`, `openai/gpt-4o-mini-transcribe`, and `mistralai/voxtral-mini-transcribe`. TTS models also exposed. Pricing is duration-based (whisper-1 at existing OpenAI rates; Voxtral Mini at ~$0.001/min). Voxtral Mini Transcribe claims to beat Whisper Large V3 and GPT-4o Mini Transcribe on multilingual tasks — Slovak support unverified (Mistral's 13-language list is not publicly enumerated).

**Relevance to us:** We call `openai/whisper-1` directly from `apps/quiz-agent`. Routing through OpenRouter gives unified billing, zero-code provider swap to Voxtral, and consistent fallback semantics — a single endpoint URL change. If Voxtral's quality holds on Slovak, it could replace Whisper-1 while staying within the same gateway already used for LLM calls. **Actionable — see issue opened this run.**

## 2. Gemini 3.5 Flash — next-gen successor to our fact-verification model (launched May 19, on OpenRouter June 12)

**Sources:**
- [Gemini 3.5 Flash — OpenRouter model page](https://openrouter.ai/google/gemini-3.5-flash)
- [Gemini 3.5 Flash pricing breakdown — pricepertoken.com](https://pricepertoken.com/pricing-page/model/google-gemini-3.5-flash)
- [Gemini 3.5 Flash complete guide — NxCode](https://www.nxcode.io/resources/news/gemini-3-5-flash-complete-guide-benchmarks-pricing-api-2026)

**Summary:** Google released Gemini 3.5 Flash on May 19, 2026. Pricing: $1.50 input / $9.00 output per 1M tokens. Beats Gemini 3.1 Pro on coding and agentic benchmarks (76.2% Terminal-Bench, 83.6% MCP Atlas) at 25% lower input cost than 3.1 Pro. Supports configurable thinking levels (minimal/low/medium/high) for cost-quality tuning. 1M token context window. Benchmark numbers are Google/third-party reported — independent verification is pending.

**Relevance to us:** We use `gemini-2.5-flash` for fact + logical verification (Tavily search → Gemini verdict). Gemini 3.5 Flash is the natural drop-in upgrade: same API surface, stronger reasoning, comparable or lower cost. Thinking-level control is directly useful — minimal for low-stakes checks, high for contested factual claims. **Actionable — see issue opened this run.**

## 3. OpenRouter Fusion — GA since June 12-13 (existing issue #15 — Evaluate OpenRouter Fusion API applies)

**Sources:**
- [Surpassing Frontier Performance with Fusion — OpenRouter Blog](https://openrouter.ai/blog/announcements/fusion-beats-frontier/)
- [OpenRouter Fusion API review — TokenMix](https://tokenmix.ai/blog/openrouter-fusion-api-review-2026)

**Summary:** Fusion moved from experimental (March 31) to fully GA. A budget panel (Gemini 3 Flash + Kimi K2.6 + DeepSeek V4 Pro) hit frontier-quality scores at roughly half the cost of a single top-tier call according to OpenRouter's own benchmark. Pricing = sum of underlying completions; no subscription. Claims are from OpenRouter's internal benchmark — treat with scepticism until independently reproduced.

**Relevance to us:** Already tracked as [#15 — Evaluate OpenRouter Fusion API as drop-in for multi-model question judge](https://github.com/michalkalis/quiz-agent/issues/15). **Watch only** — GA status means it's now production-ready to test; the existing issue covers the next step.
