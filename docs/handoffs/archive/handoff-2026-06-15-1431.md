## Decisions

- **`.serverError` message left un-wrapped** (both in `NetworkError` and where it surfaces) — the backend returns already-localized copy via `session.language`; wrapping would double-localize.
- **`handleError` fallbackMessage localized at the call site, not the combination.** The combined string is `"\(fallbackMessage): \(error.localizedDescription)"` — joining two already-localized parts with `": "`. Wrapping the *combination* would produce a useless `"%@: %@"` catalog key with no translator context, so only the prefix literal is localized and the join stays a plain concat.
- **`Language.nativeName` is NOT localized.** A language's own-language name ("Slovenčina") is identical regardless of UI language; also `Text(variableString)` (used in the pickers) doesn't auto-localize anyway. Only `Language.name` (the English exonym shown as the picker subtitle via `Text(language.name)` in `LanguagePickerView`) is localized.
- **Age-appropriate options deferred to 56.4.** `Config.ageAppropriateOptions` has no `Views/` consumer (the age picker isn't wired up), matching the 56.1b finding that `ageAppropriateDisplayName()` has zero callers. Don't localize them in 56.3b.
- **`AppErrorModel.swift` rewritten via full-file Write**, not 28 individual edits — lower risk of partial/mismatched edits since the file was read in full first.
