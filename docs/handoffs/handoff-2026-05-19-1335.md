## Decisions

**Merge approach (batch MD + generated JSON) namiesto scored JSON:**
Scored files (`data/scored/scored_2026-05-15_general_batch0XX.json`) sú review artifacts — majú IDs (`gen031_q08` atď.) a recommendation=approve, ale nemajú stable mapping na generated array indexy (batch 31 IDs nemerajú 1:1 na gen[i], a batches 32/33 majú len `question_preview` summary, nie plný text). Generated files (`data/generated/claude_batch_0XX.json`) majú plnú Question dátu (correct_answer, alt_answers, source_url, etc.) ale chýba im `id`. Batch markdown files (`data/questions/batch-0X-general.md`) majú ID + full **Q:** text — text exact-matches na 23/23/8 entries v príslušnom generated batch JSON. Riešenie: matchnúť MD entries → generated by question text, prevziať plnú Question dátu, priradiť MD-cué ID. Skript v `/tmp/claude/build_merged.py`.

**Script bug fixnúť, nie obísť:** `generate_questions_claude.py` má 2 miesta `meta.get(...)` ktoré padajú na `GenerationProvenance` Pydantic objekte. Mohol som strippnúť `generation_metadata` z merged JSON (a meta by bolo None), ale fix je správny — pre budúce importy bude funkčný.

**Stratená dup-detekcia API key:** Lokálny approve script `ChromaDBClient.add_question` potrebuje `OPENAI_API_KEY` (embedding generation). Memóorie `feedback_secrets_management.md` hovorí secrets v `.env` (gitignored), nie `~/.zshrc`. Pridal som .env loader directly do scriptu (`/tmp/claude/approve_general.py`) — robí to to čo by `python-dotenv` urobil, bez závislosti.

**Prod push autorizovaný explicitne:** Per `feedback_backend_auto_deploy.md`: ChromaDB data migrations vyžadujú ASK. Spýtal som sa cez `AskUserQuestion` pred POSTom, dostal som `Yes, push to prod`.
