# Issue 47: Upgrade GitHub Actions to Node.js 24-compatible versions

**Triage:** enhancement ¬∑ ready-for-agent
**Status:** New ‚Äî created via /triage on 2026-06-08
**Created:** 2026-06-08

## Context

GitHub is deprecating Node.js 20 as the runtime for GitHub Actions:
- **2026-06-16:** Node.js 24 becomes the default ‚Äî Node.js 20 actions run in compatibility mode
- **2026-09-16:** Node.js 20 removed from runners ‚Äî Node.js 20 actions break

The `ios-release.yml` TestFlight workflow surfaced the warning on 2026-06-08. Three actions were flagged; audit reveals the full scope spans all 4 CI workflows.

## Affected actions

| Action | Current version | Occurrences | Workflows |
|--------|----------------|-------------|-----------|
| `actions/checkout` | `@v4` | 9 | all 4 workflows |
| `actions/upload-artifact` | `@v4` | 5 | ios-ci, backend-ci, ios-release, web-ui-ci |
| `webfactory/ssh-agent` | `@v0.9.0` | 1 | ios-release |
| `actions/setup-node` | `@v4` | 3 | web-ui-ci |
| `actions/cache` | `@v4` | 1 | ios-ci |
| `actions/setup-python` | `@v5` | 4 | backend-ci |

> *Note: Only `checkout@v4`, `upload-artifact@v4`, and `ssh-agent@v0.9.0` were explicitly flagged in the deprecation warning. The others (@v4/v5 actions) should be verified ‚Äî they may already ship Node.js 24 support via their floating tags.*

## Deadline

**2026-06-16** (8 days) ‚Äî before this date, set `FORCE_JAVASCRIPT_ACTIONS_TO_NODE24=true` in workflows as an interim safeguard if upgrades aren't done.
