## Notes & deviations from brief

1. **`question.state` probe missing.** The hidden Text element with id `question.state` (added in `888dba1`) was removed by the WIP refactor in `QuestionView.swift`. The state name is now exposed via `accessibilityValue` on `question.statusPill`, which surfaces in `snapshot_ui` as `AXValue`. The runner adapted to read `AXValue` of `question.statusPill` instead of `AXLabel` of `question.state`. This works for RS-01 (the state name is captured) but the regression-scenarios.md doc and issue-18 brief should be updated, OR the WIP should restore the dedicated probe.
2. **Mock STT auto-emits the committed transcript** when recording starts in `--ui-test` mode (preview answer = "Paris"). The `curl committed` step is therefore idempotent for RS-01 — it confirms the HTTP listener path is wired but isn't strictly required for this scenario to pass. A scenario that explicitly tests the HTTP injection path would need a mock that doesn't auto-emit, or a longer recording window before injection.
3. **Step ordering matters for log capture.** `start_sim_log_cap({captureConsole: true})` relaunches the app *without* arguments, dropping `--ui-test`. Correct order: `launch_app_sim({args: ["--ui-test"]})` first, then `start_sim_log_cap({})` with no `captureConsole` flag (structured logs only).

## Followups (separate from RS-01 verdict)

1. **Bug: invalid transition + error after auto-confirm.** ~10s after the confirmation sheet appears, the auto-confirm timer fires `resubmitAnswer(_:suppressAudio:)` which attempts a `processing → processing` transition (rejected by `validTransitions`) and then errors with `❌ No evaluation in response, cannot show result`. Two issues:
   - `resubmitAnswer` should not be the path used for non-edited auto-confirm. The user did not edit the answer.
   - `processing → processing` is rejected by the legal-transition table — the caller should be in a different state when resubmit fires, OR `processing → processing` should be allowed as a no-op.
   - Either way, the mock-network response not containing an evaluation should not produce a user-facing error in `--ui-test` mode.
   - File as RS-XX or as a separate bug — does not block RS-01.
2. **WIP files diverge from regression doc contract.** Restoring the hidden `question.state` probe in `QuestionView.swift` is one line of code and would let `regression-scenarios.md` stay accurate. Either restore or update the doc to use `question.statusPill.accessibilityValue`.
3. **Sentry SentryCppHelper.o ld warning** (`found architecture 'arm64', required architecture 'x86_64'`) is unrelated but noted.
