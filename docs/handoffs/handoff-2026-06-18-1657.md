## Decisions

- **AuthService degrades gracefully (Optional returns, never throws).** Required for the staged rollout: prod returns 503 until `AUTH_JWT_SECRET` + migration are live, so the app must work *without* a bearer via the legacy `user_id` grace path. `accessToken()`→nil is the contract that makes that fallback work.
- **Re-bootstrap on *any* refresh failure is safe.** If the backend is merely offline, the re-bootstrap also returns nil → no orphan identity minted. An orphan is only possible when the backend is reachable *and* the refresh is specifically rejected — which is exactly the legitimate re-bootstrap case.
- **`-1011` flake root cause (two bugs in my own test harness, both fixed):** (1) AuthServiceTests' `defer` was clearing `StubURLProtocol.handler` (NetworkServiceTests' protocol) instead of `AuthStubURLProtocol.handler` — wiping the other suite's handler mid-flight under cross-suite parallelism. (2) `Thread.sleep` in the single-flight stub ran on the shared URL-loading thread, starving the concurrent `downloadAudio` request. Fix: corrected the `defer` target + moved the handler off the loading thread via `DispatchQueue.global().async`. `.serialized` only orders tests *within* a suite, not across suites — that's why separate static handlers were necessary.
- **downloadAudio left unauthenticated** — it uses the continuation/dataTask path and the audio endpoint doesn't gate on identity. Documented, intentional.
- **`PrivacyInfo.xcprivacy` unchanged** — Keychain is not on Apple's required-reason API list, so no manifest entry needed.
