## Decisions

- **The fix lives in the generator, not the orchestrator.** The orchestrator stage only has the *global* fact list — which is exactly *why* it did the wrong single-URL fallback. The disjoint per-sub-batch fact slice (`_partition_facts`) is in scope only inside the generator, so per-question matching belongs there. The orchestrator fallback was demoted to a last-resort net, not removed (F8 still needs it for fact-free sub-batches).
- **Deterministic content-token match — no LLM, no schema/prompt change.** Matches the model's `source_excerpt` (classic/text path) or the question+answer+explanation (MCQ structured path, whose schema carries *no* source fields at all) against each fact's text+excerpt. Honors the backend rule "don't use the model for deterministic transforms"; smaller footprint than adding a fact-id to the structured output + prompt.
- **F8 preserved (fail-loud kept).** Every factual question still ends with a real `source_url`; the fix changes *which* URL, never *whether* one exists. Gap-only fill (a model-emitted url is kept); no-match → the slice's first sourced fact (still slice-local, never the global pack head).
- **No deploy.** Change is dormant (generation parked, no migration). Deploy alongside the un-park, not before.
- **Left a pre-existing unused `import uuid` (F401) in `advanced_generator.py` untouched** — not introduced this session, out of scope per minimal-footprint.
