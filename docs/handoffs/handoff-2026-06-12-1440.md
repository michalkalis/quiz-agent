## Decisions

**Fresh-Task handoff pattern (54.5):** `confirmAnswer()` calls `cancelAutoConfirm()` which cancels the Task it runs inside. Rather than restructuring the cancel-chain or using `withTaskCancellationHandler`, we hand off to a fresh `Task { await self.confirmAnswer() }` at the end of the auto-confirm Task. The fresh Task is not under the auto-confirm key so cancellation doesn't affect it. This is the minimal fix that keeps manual-confirm and auto-confirm on the same submit path (matching existing #19 fix intent).

**`activeErrorModel` as `@Published private(set)` (54.15):** `ContentView` needed the full `AppErrorModel` at render time but the `.error` state only carries `Error? + ErrorContext`. Adding the field to the VM (set in `setError`) avoids changing the `QuizState` enum shape or threading the model through state, and keeps `setError` as the single entry point. Trade-off: one new `@Published` field drifts `.stableDump` snapshots — expected, handled by re-recording 2 baselines.

**Snapshot drift is pure model drift (confirmed):** `correctVariant.1.txt` and `incorrectVariant.1.txt` both have `_errorMessage` at line 96 but no `activeErrorModel` entry → adding the field inserts a new line in the dump. These 2 baselines are safe to re-record; the 3 known stale ones (HomeView idle / QuestionView asking / recording) are explicitly deferred until UI-final (umbrella plan).
