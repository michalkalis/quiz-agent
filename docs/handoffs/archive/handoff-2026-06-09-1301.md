## The 8 review themes (from the artifact)

1. **CRIT — Pink means both "good" and "bad."** `accent-pink #FF3D8F` is brand + CTA + points (positive) AND error/incorrect (negative). Fix: reserve pink for brand/CTA, use red `#FF4444` for errors, green for correct.
2. **HIGH — Voice-first affordance visually subordinate.** (✅ ADDRESSED for question screens.)
3. **HIGH — Punchy English headlines don't localize** (NAILED IT / CLOSE—BUT NO / OOPS). Primary user is Slovak.
4. **HIGH — Scoring model illegible** (points vs "87/100" vs "best 9.5" vs "avg 8.7" vs streak — no clear units/relationship).
5. **MED — Two contradictory "Listening" states.** (✅ ADDRESSED — unified into one-state model.)
6. **MED — Branding redundancy** (double wordmark on Home: "hangs." + giant "HANGS"; casing inconsistent).
7. **MED — Onboarding & Paywall have NO Pencil design** despite existing in code — most important screens for retention/monetization.
8. **NIT — Glanceability while driving** (text-heavy results; "Speak scores aloud" defaults OFF — should be ON for hands-free).

## Decisions

- **Voice-first question screens: mic circle REMOVED per user feedback.** User rejected the big central mic visualizer — wanted the QUESTION to stay dominant (full text, can wrap), and only a minimalist indicator at the bottom. Final layout: dominant question (Anton 40px) → minimal mini-waveform + status label in the middle → bottom control row = primary **Záznam/Zastaviť** button (two states: record ↔ stop) + compact **Preskočiť** (icon+text, shrunk, right of main button).
- **Live transcript: moved to bottom, de-emphasized.** Per user: transcript card sits above the buttons, restyled to border-only (no white fill, no shadow), text in secondary color.
- **Copy localized to Slovak + made truthful.** Removed fake voice-command prompts ("povedz 'stop'" / "povedz 'preskoč'") since commands don't exist; replaced with silence-model copy ("Keď dohovoríš, odpoveď sa odošle sama").
- **Design system facts:** fonts = Anton (display), Inter (body), IBM Plex Mono (mono labels). pink `#FF3D8F`, purple `#8B5CF6`, blue `#0A84FF`, red `#FF4444`, green `#22C55E`. Light/dark themed. Radius/spacing/size tokens all defined as variables — always use `$token` refs, never hardcode.
