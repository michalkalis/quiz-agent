## Decisions

- **Pencil `batch_design` call shape (IMPORTANT — cost a few failed calls to find):** the operations string must be passed under a parameter named **`input`**, NOT `operations` (despite the advertised JSON schema requiring `operations`). The advertised schema requires `operations`, so pass BOTH: a dummy `operations` value (e.g. `"noop"`) to satisfy client-side validation AND the real ops under `input`. Operation functions use **full names** — `Update(...)`, `Insert(...)`, `Delete(...)`, `Copy(...)`, `Replace(...)`, `Move(...)` — NOT the single-letter `U()`/`I()` forms shown in the tool description (those throw `ReferenceError: 'U' is not defined`). The prior handoff's "U() ops" note is wrong/stale.
