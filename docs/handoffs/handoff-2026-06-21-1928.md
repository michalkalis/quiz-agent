## Decisions

- **`method="function_calling"`, not the langchain `json_schema` default** in `with_structured_output`
  — gateway-safe: under `LLM_GATEWAY=openrouter` the model id becomes `openai/gpt-4o`, and OpenRouter
  proxies function-calling reliably but `json_schema` proxying is not guaranteed.
- **Extracted `_build_batch_prompt` + `_finalize_questions`** rather than duplicating ~80 lines into the
  structured path — DRY, no drift. Behavior-preserving (classic best-of-N + `_parse_response` untouched;
  163 tests confirm).
- **42.29 kept the `min_quality_score` param** even though its only use (the deleted Stage-4 stub) is
  gone — it's part of the public order API (`routes.py:105` ← `schemas.py:94`). Removing it is out of scope.
- **42.26 must be implemented as SYNC for DedupStage** — `QuestionStore` Protocol
  (`question_store.py:51`) declares `find_duplicates` sync, and `DedupStage._is_cosine_duplicate`
  (`dedup.py:93`) calls it synchronously inside an `except: return False` (so an async method called
  sync = **silent pass-through**, dedup silently disabled). **Do NOT use a separate sync DB engine**
  (the blueprint's first idea) — the repo already has the right pattern: `_BackgroundLoop`
  (`run_coroutine_threadsafe`) in `apps/quiz-agent/app/retrieval/sync_pgvector_store.py`, whose
  `find_duplicates` is a deliberate `NotImplementedError` stub. **42.27 plan:** move `_BackgroundLoop` +
  `SyncPgvectorStore` to `packages/shared/quiz_shared/database/` (quiz-pack-api can't import from
  quiz-agent), implement its `find_duplicates` to bridge to the new async one, re-point quiz-agent's
  import, then `app/worker/worker.py:52` wraps `PgvectorQuestionStore` in that sync facade for
  DedupStage; add `--dedup-store pgvector` to `generate_pack.py` (default stays `_NoopQuestionStore`).
- **Did NOT run `ruff format` on the whole `advanced_generator.py`** — it over-reached and reformatted
  dozens of unrelated pre-existing lines (quote/slice/wrap churn). Reverted that and re-applied only my
  edits. `apps/quiz-pack-api/` is **not** in CI's ruff scope (`backend-ci.yml` lints only
  `apps/quiz-agent/` + `packages/shared/`), so hand-clean code is sufficient there. The pre-existing
  `F401 uuid` unused import in that file is not mine — leave it.
