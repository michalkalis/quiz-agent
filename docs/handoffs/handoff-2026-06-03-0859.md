## Decisions / constraints

- **True/False is NOT a new type** — it's a 2-option `text_multichoice`. Don't add an enum case; treat as an MCQ visual variant. (Confirmed: no `true_false` case in iOS, matches backend `pattern_routing.py` which routes `true_false` → `text_multichoice`.)
- **Backend already supports spoken MCQ answers** — `_evaluate_mcq` (`apps/quiz-agent/app/evaluation/evaluator.py`) normalizes and matches key *or* value, 1.0/0.0. No backend change needed for Track E. See `docs/reference/question-types.md`.
- **Two color layers exist** — new screens use `Theme.Hangs.Colors` (light-only). Port dark there rather than reviving the old `Theme.Colors` layer, to avoid mixing. Flag the duplication for later consolidation; don't fork silently (CLAUDE.md rule 7).
- **API contract** — these changes are UI-only; no Pydantic/Codable changes expected. If you do touch models, run `/verify-api`.
- **#F6F7F9 not yet in code** — closest existing is old-layer `Theme.Colors.bgSecondary` `#F8F9FA` (light-only); add the new value to the Hangs layer.
