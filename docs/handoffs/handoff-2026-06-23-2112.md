## Why the wrapper exists (the problem it solves)

`overnight.sh` always cuts its work branch from **main HEAD** and has **no resume knob**; the worker picks
the next task purely from the **focus file's checkboxes**. Local main is pristine `00a434d` (no #72
progress). The Claude session limit caps a max-effort Opus run to ~**3–4 iterations per 5h window**, so #72
must span several windows. Without the wrapper, every scheduled fire would cut from pristine main and
**restart at P0.1 → zero net overnight progress**. The wrapper advances LOCAL main to the cumulative
`ralph/*` tip first, so the worker resumes at the next open box. `origin/main` stays at `00a434d` (founder
gate); only `ralph/*` branches are ever pushed.
