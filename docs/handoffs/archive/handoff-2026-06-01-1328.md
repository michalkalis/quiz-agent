## Decisions

- **Full tokenization now, not just backgrounds** — doing only backgrounds would make dark mode visually broken (text stays dark on dark bg). Approved by user mid-session. Folds handoff's "Part 4 token work" into Part 1.
- **`accent-pink = #FF3D8F`** (not `#FF4FB6`) — this was the dominant pink in all 8 screens; `#FF4FB6` was a stale token value used only in components. Unified to `#FF3D8F`. Approved by user.
- **White knobs/icons on accent backgrounds → `$text-on-accent`** — toggle knobs, waveform bars, icons on pink/green buttons all bind to `$text-on-accent` (always `#FFFFFF`) so they stay white in both modes, not `$bg-card` (which inverts in dark).
- **`replace_all_matching_properties` is broken for variable bindings** — writes `\$token` not `$token`; never use it for binding rewrites. Use `batch_design U()` only.
- **Translucent decorative fills stay hardcoded** — `#FF3D8F14`, `#FF3D8F1A`, `#FF3D8F33` pulse rings and badge backgrounds are intentionally left as hex; they're decorative overlays that look fine in both modes.
