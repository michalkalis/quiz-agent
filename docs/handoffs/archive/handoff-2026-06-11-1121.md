## Decisions

- **OpenRouter over LiteLLM / Vercel / Portkey** — LiteLLM unifies the *code* interface but not billing (keeps N provider bills); managed gateways are overkill for solo. OpenRouter is the only one that delivers one prepaid balance + one invoice.
- **Credits, not BYOK** — BYOK ("bring your own key") is cheaper (free routing < 1M req/mo at this scale) but keeps three provider bills, so it fails the stated goal. Credits = one bill, ~5.5% top-up fee; simpler *and* goal-satisfying.
- **All-in scope, but staged + verified** — migrate everything incl. Whisper/TTS/embeddings/`gpt-image-1`, but prove text first and verify each model serves cleanly via OpenRouter. Any layer OpenRouter can't serve cleanly stays on direct OpenAI and is flagged loudly (fail-loud, no forcing).
- **Central client factory is the proper fix** — not scattered `base_url` edits. ~15 call sites across two apps currently hardcode models/keys; the factory is valuable independent of OpenRouter.
- **Waited for the key rather than starting Phase 2 early** — founder chose to do spike→factory→cutover in one clean pass to touch the ~15 files once instead of twice.
