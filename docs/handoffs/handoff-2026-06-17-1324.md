## Decisions

- **Recommended approach:** anonymous-first **self-issued JWT** on the existing FastAPI/Fly.io+Postgres
  stack (no managed auth provider) — chosen for global portability (no region/vendor lock-in) and
  zero incremental cost. `ElevenLabsTokenResponse` is the existing short-lived-token blueprint.
- **App Attest → Phase 1** (founder chose strong anti-abuse now): fake clients can't mint the
  anonymous token. Bumps Phase 1 from M to **L** (~+1 week).
- **iOS-only MVP** → Sign in with Apple only; **email magic link deferred** (web/Android only after
  iPhone succeeds). Architecture must stay token-provider-agnostic so magic link is cheap to add.
- **Paid IAP can launch with no login** — verified vs Apple docs (WWDC22 / TN2413 / Guidelines
  §3.1.1, §5.1.1(v)): subscriptions + non-consumables restore via Apple ID with no app account;
  **consumables do NOT restore**. → **#50 constraint:** ship premium as an auto-renewable
  subscription (packs, if any, as non-consumables — never consumables).
- **EU residency already satisfied:** both Fly apps `primary_region = "cdg"` (Paris). Add US regions
  on expansion, no auth-vendor change.
- **Corrected two #58 assumptions:** (1) the `quiz-pack-api` JWS verifier
  (`apps/quiz-pack-api/app/storekit/verifier.py`) *does* exist but is StoreKit-IAP-only — not
  reusable for general auth; (2) `POST /api/v1/usage/{userId}/premium` is currently
  **unauthenticated** — anyone can self-grant premium (Phase 1 fixes this).
- File placement: deliverable went to `docs/research/` (typed home per the new File-Placement rule),
  not `docs/product/` as the original #58 text suggested.
