## 1. OpenRouter subagent + advisor compound-routing tools

**Source:** [OpenRouter cookbook — Build a Token-Efficient Review Agent with Advisor](https://openrouter.ai/docs/cookbook/building-agents/advisor-server-tool) · [OpenRouter docs — MCP Servers](https://openrouter.ai/docs/guides/coding-agents/mcp-servers)

Released as part of the June 12–13 OpenRouter platform wave alongside Fusion. Two server tools, separate from Fusion's fan-out pattern:

- **`openrouter:subagent`** — delegates a self-contained subtask (summarize, extract, reformat) to a cheaper worker model mid-generation. Worker has its own tool loop; only the final text returns to the orchestrator. Billed at worker-model rates. 10 delegations/request cap.
- **`openrouter:advisor`** — lets a cheap orchestrator call a stronger model for hard sub-decisions or sanity checks. Escalates only when needed; routine tokens stay at the cheap model's rate.

**Relevance to us:** The advisor pattern maps directly to our real-time answer-evaluation path (currently `gpt-4o` for every answer, including unambiguous ones). A fast cheap model (e.g. `gpt-4o-mini`) handling routine yes/no judgments and escalating to `gpt-4o` only for edge cases could cut latency and cost on that hot path. The subagent pattern overlaps Fusion territory (issue #15 — "Evaluate OpenRouter Fusion API as drop-in for multi-model question judge") but works at the call level rather than as a separate API. **Supplement to issue #15; no separate issue needed — mention the advisor pattern when evaluating Fusion.**

## 2. NVIDIA Nemotron 3.5 ASR — open-weight 40-locale streaming STT

**Source:** [MarkTechPost — Nemotron 3.5 ASR release](https://www.marktechpost.com/2026/06/06/nvidia-releases-nemotron-3-5-asr-a-600m-parameter-cache-aware-streaming-model-transcribing-40-language-locales-in-real-time/) · [Hugging Face model card](https://huggingface.co/nvidia/nemotron-3.5-asr-streaming-0.6b) · [LiveKit blog](https://livekit.com/blog/nemotron-3.5-asr-multilingual-teleprompter)

Released June 4 on Hugging Face (OpenMDW-1.1 license). 600M-parameter cache-aware streaming Parakeet+RNN-T encoder. Key specs: 40 language-locales in a single model, configurable chunk sizes (80 ms–1 120 ms), automatic language detection, punctuation and capitalisation. At 80 ms chunks: ~17× more concurrent streams vs. buffered approaches. Slovak is a plausible inclusion in a 40-locale Slavic-language pack but is unverified.

**Relevance to us:** Our Slovak STT path uses cloud `whisper-1` via the backend (issues #14, #16 track cloud alternatives). Nemotron would require self-hosting (GPU needed) or integration via Together AI, which carries it. Not a near-term swap, but if Slovak is confirmed and self-hosting proves feasible, it could serve as a fallback for privacy-sensitive or offline use cases. **Watch only — verify Slovak locale before considering further.**
