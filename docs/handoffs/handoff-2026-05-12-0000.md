## Decisions

- **Duck, don't pause music.** Apple HIG explicitly recommends `.duckOthers + .interruptSpokenAudioAndMixWithOthers` for "apps with spoken audio prompts" (navigation/workout category). Full pause is jarring and inconsistent with Waze/Maps UX.
- **`interruptSpokenAudioAndMixWithOthers` pauses podcasts automatically** — this is correct and expected (two voices = unintelligible). No user-visible change needed.
- **Do NOT per-prompt activate/deactivate session** (Apple HIG ideal) — the `SpeechAnalyzer` / `AVAudioEngine` setup between questions makes the state too complex. Activate once at quiz start, deactivate at quiz end.
- **Category switch is scoped to `performPlayback`, not the ViewModel** — single chokepoint, caller code unchanged.
- **Defer restore must use real error logging**, not `try?` silent swallow — if restore fails, next recording session breaks silently. Sentry breadcrumb mandatory.
- **Both prior plans had the same gap:** the TTS-volume plan restores `.playAndRecord + .mixWithOthers` after TTS (recording phase has no ducking), the UX-research plan ignores `.playAndRecord` attenuation. The unified plan fixes both by changing the defaults AND adding the per-TTS category switch.
