## Decisions

- **Endpoints fail safe with 503, not a half-secured boot.** `/auth/*` is only wired when `DATABASE_URL` AND a ≥64-char `AUTH_JWT_SECRET` are both present; `TokenService` raises on a too-short secret (fail loud — a misconfigured prod must not boot half-secured).
- **A present-but-invalid bearer → 401, never a silent fall-through to the grace path** (that would be an auth downgrade). The body-`user_id` fallback fires only when there is *no* bearer header at all, and only while `LEGACY_USER_ID_GRACE` is on (read live per-request, default on).
- **`_touch_identity` never downgrades an existing identity to legacy** — on conflict it only bumps `last_seen_at`. A real bootstrap id passed in the body stays non-legacy.
- **Bootstrap rate limit = `20/minute` on `Fly-Client-IP`** — generous on purpose (household NAT shares one IP; a failed refresh re-bootstraps through here). App Attest (Part B) is the real anti-abuse lock, not the IP limit.
- **`setPremium` admin-key guard left untouched** and pinned with a new HTTP-level test so a future refactor can't quietly route it through the bearer middleware (§10 correction).
