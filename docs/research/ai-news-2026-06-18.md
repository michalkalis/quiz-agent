## 1. GPT-Realtime-2 (and companion models) — still uncovered since May 7

**Sources:**
- [Advancing voice intelligence with new models in the API — OpenAI](https://openai.com/index/advancing-voice-intelligence-with-new-models-in-the-api/)
- [OpenAI GPT-Realtime-2: new voice models for AI agents — Heyloha](https://www.heyloha.ai/en/blog/openai-gpt-realtime-2)
- [OpenAI launches new voice intelligence features — TechCrunch](https://techcrunch.com/2026/05/07/openai-launches-new-voice-intelligence-features-in-its-api/)

**Summary:** OpenAI shipped three new Realtime API models on May 7, 2026. `gpt-realtime-2` is a GPT-5-class speech-to-speech model with configurable reasoning for production voice agents; `gpt-realtime-whisper` is its streaming-STT companion (lower latency than `whisper-1`); `gpt-realtime-translate` handles streaming speech translation across 70+ input languages / 13 output languages. The Realtime API is now GA with MCP server support, SIP/phone calling, and image input. Two new TTS voices (Marin, Cedar) added. Slovak support in `gpt-realtime-translate` unverified — 13 output languages are not publicly enumerated.

**Relevance to us:** We currently run a STT→LLM→TTS pipeline (`whisper-1` → `gpt-4o` → `tts-1`). `gpt-realtime-2` collapses this into a single speech-to-speech roundtrip, which could halve perceived latency in the in-car quiz loop. However, this is a significant architectural change, and Slovak support in the speech path is unconfirmed. Already evaluating Voxtral and OpenRouter Audio gateway via issues [#14 — Evaluate MAI-Transcribe 1.5 vs Whisper-1 for Slovak STT](https://github.com/michalkalis/quiz-agent/issues/14) and [#16 — Evaluate OpenRouter Audio API as unified STT gateway and benchmark Voxtral for Slovak](https://github.com/michalkalis/quiz-agent/issues/16). **Watch only** — revisit after #14/#16 resolve Slovak accuracy baseline; then decide whether a speech-to-speech pivot is worth scoping.
