## Decisions

- **LaunchAgent kickstart is the only auth-safe launch path on mba.** `ssh mba → claude` returns "Not logged in" (Keychain in gui domain only). All tmux-from-ssh launchers (`launch-issue44.sh` etc.) would have failed on auth. The fix was to use `launchctl kickstart gui/<uid>/com.quizagent.ralph-overnight` which runs in the gui domain where claude is authenticated. Implication: future manual iOS runs must also be kicked from the LaunchAgent or the GUI Terminal on mba, not from a raw ssh tmux session.
- **XcodeBuildMCP fix confirmed server-side but NOT end-to-end.** The MCP `initialize` handshake with `xcodebuildmcp@2.6.2` succeeded on mba. But the current live beh started at `672d476` (before the fix on `3e33b54`) so #44/45 iterations ran without XcodeBuildMCP and deferred screenshot-verify to human. The **next gui-domain iOS beh** (after this run's branch is merged and mba pulls main) will be the first to actually invoke `mcp__XcodeBuildMCP__screenshot` — look for it in the iter log to confirm end-to-end.
- **`--mcp-config` gated to iOS focus files only.** Backend focus files (#51, #30, #42) don't need the npx server; gating avoids a ~10s startup penalty per backend iteration.
