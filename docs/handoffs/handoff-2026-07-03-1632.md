## Decisions

- **Fix the reset at the merge, not only at delete:** the reviewer proposed rebinding the App Attest key to the user id; rejected — after account delete that would brick re-bootstrap (key pointing at a deleted subject). Keeping the anon's usage rows frozen + carrying today's count back on delete closes both the sign-out and the delete loop without touching the key binding. No migration needed.
- **Only *today's* counter is carried back on delete, without premium** — GDPR-minimal fraud prevention (GREATEST, can only tighten); full-history copy would effectively retain account data past erasure. Premium entitlement dies with the account.
- **`/auth/logout` is always 204** — idempotent, never reveals token validity.
- **Accepted risk (documented, not fixed):** `require_auth` in `apps/quiz-agent/app/api/deps.py:321` returns `authenticated=False` instead of rejecting when `LEGACY_USER_ID_GRACE` is on — every current call site handles it, but the name invites misuse. Belongs to #65 (endpoint auth hardening); note it there rather than churning call sites now.
