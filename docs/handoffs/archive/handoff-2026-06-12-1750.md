## Decisions

- **54.13 bucket semantics:** correct = `.correct` only, incorrect = `.incorrect` only; `.partiallyCorrect`/`.partiallyIncorrect`/`.skipped` land in neither bucket. Within the founder's resolved "show fractional as-is" decision; guarantees count consistency by construction.
- **54.16 reference type over `@State Task`:** a value-type `@State Task<Void, Never>?` outside an installed view is a constant binding — writes don't persist, so onChange cancelled nothing and tests flip-flopped with parallel-test timing. `@MainActor final class MCQDelayedSubmit` held in `@State` shares one instance across struct copies and made the guard deterministically unit-testable (per `feedback_proper_solutions` — structural fix, not a patch).
- **TSan timing:** fixed-sleep assertions are flaky under the test scheme's thread sanitizer (coarse timer resolution) — the fires-once test polls up to ~2s instead of a fixed 200ms wait. Pattern to reuse in future timing tests.
