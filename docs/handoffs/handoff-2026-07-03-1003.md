## Findings so far (Phase 1 evidence — keep, this is the core value)

Confirmed on sim:
1. **Settings back button top-right** (custom `arrow.left`, id `settings-back-button`) — HIG violation; and **edge-swipe back does nothing** (screen isn't a NavigationStack push; whole header incl. back button also scrolls away, leaving no visible navigation when scrolled). `05-settings-top.png`, `06-settings-bottom.png`.
2. **Home selectors** (Language/Difficulty/Categories) are plain SwiftUI Menus **without a checkmark on the current selection**; "Categories" is single-select despite the plural label. `02–04-*.png`.
3. **Replay control exists but looks missing**: on the open-answer quiz screen it's a small blue text link "▶ replay question" (id `question.replay`) — founder believed it doesn't exist. **No mute/TTS-off control anywhere.** `11-quiz-question.png`.
4. **MCQ vs open-answer top bars are two different designs** (MCQ: "ADULTS · QUESTION 1", no timer pill, no progress bar, full-width "Skip question"; open: pink lowercase category, "ANSWER 18s" pill, thin progress bar, circular "Skip"). Directly relevant to founder's "horná časť" doubt. `16-quiz-mcq.png` vs `11-quiz-question.png`.
5. **Suspected state bug (P1, needs /diagnose or code read)**: typing into "Type answer instead" while auto-record was live produced a confirmation with the *voice* transcript; afterwards the session got stuck — result header showed `01 / 10` while answering Q2 (off-by-one), the same question repeated indefinitely, streak grew (→5) while SCORE stayed 1. Clean runs progress fine (Q1→Q2 ok), so trigger = mixed text+voice mid-state. Repro steps in shots `14`, `15`. Caveat: observed under `--ui-test` mocks; verify against real backend before filing as prod bug.
6. **Result screen auto-advance is 7s** with tiny "Stay here" link — hard to hit even by automation; for a driver it's also the only way to linger. `15-quiz-result-correct.png`.
7. **"Why is this correct?"** button appears on some results only (Q1 capital-of-France; not sulfuric/planet results) — probably shown only when the fixture question has an explanation; verify in `ResultView.swift` (`result-why-correct-button`).
8. **Answer countdown keeps ticking while typing** (was at 3s when submitting "London") and **keeps running behind the End Quiz dialog**.
9. **End Quiz dialog has no Cancel button** — single red "End Quiz" in a custom card; dismiss-by-tap-outside only. `21-quiz-end-dialog.png`. Ending mid-quiz goes straight Home (no partial summary).
10. **Sign in with Apple**: sim has no Apple ID (system alert only, flow not exercisable). Founder's "name didn't show after re-sign-in" is the classic ASAuthorizationAppleID trap — fullName/email are delivered **only on first consent**; app must persist them then. Verify persistence in the auth/account code (search `ASAuthorization` / `fullName`).
